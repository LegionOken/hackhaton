Для реализации проекта Bozor-Analitika, как AI Engineer, вам нужно сосредоточиться на
  создании конвейера данных (data pipeline) и подборе моделей для обработки рыночной
  информации.

  Вот основные направления, которые необходимо интегрировать:

  1. Сбор и парсинг данных (Data Ingestion)
  Поскольку вы работаете с реальными ценами и спросом, вам нужны структурированные данные.
   * Задача: Извлечение информации из Telegram-каналов, сайтов объявлений (OLX и др.),
     прайс-листов поставщиков (PDF/Excel).
   * AI решение:
       * LLM (GPT-4o, Claude 3.5 Sonnet) или Local LLM: Для извлечения сущностей (название
         товара, цена, объем, регион) из неструктурированного текста.
       * OCR: Если данные приходят в виде скриншотов или фото ценников.

  2. Очистка и нормализация (Data Normalization)
  Цены пишутся по-разному, названия товаров отличаются.
   * Задача: Сведение товаров разных поставщиков в одну категорию (например, "Samsung S24" и
     "S24 256GB" — это один товар).
   * AI решение:
       * Semantic Matching: Использование векторных вложений (Embeddings, например
         text-embedding-3) и векторных баз данных (Pinecone, Weaviate, pgvector) для
         классификации товаров и поиска похожих.

  3. Аналитика и прогнозирование (Predictive Analytics)
   * Задача: Определение рыночной стоимости и прогнозирование спроса.
   * AI решение:
       * Time Series Analysis (XGBoost, Prophet): Для прогнозирования цен на основе
         исторических данных.
       * Anomaly Detection: Автоматическое обнаружение подозрительных ценовых скачков.

  4. Рекомендательная система (Matching Engine)
   * Задача: Предложение "лучших предложений" для B2B клиентов.
   * AI решение:
       * Hybrid Matching: Сочетание бизнес-правил (логистика, наличие) с ранжированием на
         основе ML-моделей (Learning to Rank).

  ---

  С чего предлагаю начать (план действий):

   1. Создайте структуру для AI-сервисов:
      В папке проекта (вне packages) создайте папку ai-services для хранения скриптов обработки
  данных.
   2. Выбор стека:
       * Python: FastAPI (для бэкенда AI-сервисов).
       * LangChain / LlamaIndex: Для оркестрации LLM-вызовов при парсинге.
       * Vector DB: Выберем PostgreSQL с расширением pgvector, так как это надежно для B2B.
